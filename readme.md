# MEDISCARA Manager Web Application

## Purpose

MMWA provides a platform for the plant manager to interact with the production
process via Production Oders.
MMWA also gives feedback about the products in real time.

## Setup

---

### Docker

KeyRock must be running, preferrably inside a docker container.

Here is a development example configuration for KeyRock in docker compose:

```docker
  keyrock:
    image: fiware/idm:7.6.0
    container_name: fiware-keyrock
    hostname: keyrock
    networks:
      default:
        ipv4_address: 172.19.1.11
    depends_on:
      - mysql-db
    ports:
      - '8088:8088'
      - '8089:8089'
    environment:
      - DEBUG=idm:*
      - IDM_DB_HOST=mysql-db
      - IDM_HOST=http://localhost:8088
      - IDM_PORT=8088
      - IDM_DB_PASS=secret
      - IDM_DB_USER=root
      - IDM_ADMIN_USER=keyrockadmin
      - IDM_ADMIN_EMAIL=kadmin@example.com
      - IDM_ADMIN_PASS=admin
      - IDM_EMAIL_HOST=mailer
      - IDM_EMAIL_PORT=25

  mysql-db:
    restart: always
    image: mysql:5.7
    hostname: mysql-db
    container_name: db-mysql
    expose:
      - '3306'
    networks:
      default:
        ipv4_address: 172.19.1.9
    environment:
      # Development use only
      # Use Docker Secrets for Sensitive Data
      - MYSQL_ROOT_PASSWORD=secret
      - MYSQL_ROOT_HOST=172.19.1.11
    volumes:
      - mysql-db:/var/lib/mysql

  volumes:
      - mysql-db: ~
```

In the Keyrock IDM create an application. This manu can be found by clicking the 'Register' button at `keyrock/idm/applications`.
Set the applcation's url to `http://localhost:8000` and the
callback URL to `http://<The server machine's IP>:8000/home`.

Override the **.env** file with the following:

- CLIENT_ID: The ID of the app, generated by the IDM
- CLIENT_SECRET: The secret of the app, generated by the IDM
- KEYROCK_URL: The URL of the Keyrock IDM
- OCB_URL: The URL of the Orion Context Broker

After this the image can be built and run by running these commands

```bash
docker build -t manager-app:1.0 .
docker run --network="host" -p 8000:8000
```

If you want to integrate the image into docker compose, append the following lines to the _docker-compose.yaml_ file.

```docker
mmwa:
  image: manager-app:1.0
  networks:
    default:
```

Or alternatively by running the following python command from the repository's root directory:

```bash
python3 -m src/manage.py runserver 0.0.0.0:8000
```

## Overview

---

The MMWA was created as a part of the [MediSCARA](!https://github.com/ppuska/mediscara "The github repository of the project") project. Developed by HSM Automatika Kft.

### Achitecture

The application was build using [Django](!https://www.djangoproject.com/), and uses [FIWARE](!https://www.fiware.org/) software componenst as a backend provider.
The POs are stored as entities in the Orion Context Broker, and are accessed by a custom FIWARE Python [API](!https:github.com/ppuska/mediscara.fiware). The authentication is provided by the [KeyRock Identity Manager](!https://fiware-idm.readthedocs.io/en/latest/).
